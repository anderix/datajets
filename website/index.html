DataJets

DataJets is an open source framework for creating database-aware classes in Java that contain create, read, update and delete functionality. It was created to bridge the gap, known as impedance mismatch, between object and relational paradigms. Its goal is to address 90% of the basic database actions that many developers find themselves coding over and over again in their applications.

DataJets aid in object-relational mapping and provides an alternative to the more complex APIs that exist today. Building basic DataJets requires no SQL and very little knowledge of JDBC, allowing developers to focus on the business logic of applications. For more advanced DataJets knowledge of SQL and relational database concepts is recommended. DataJets on their own make good business objects for many solutions, and including them in your project promotes modular architecture and a seperation of concerns.

There are several ways to use DataJets, ranging from simply instantiating the existing classes for connecting to database tables to inheriting from the base classes to create more complex architectures with support for connection pooling. DataJets can encapsulate the logic for creating corresponding database tables, eliminating the need for maintaining seperate SQL statements for the installation of your applications.

DataJets is an approachable framework for developers. Writing a basic application to access a database requires just a few lines of code. The existing classes include methods for creating, reading, updating and deleting database records, and can be extended to scale to much more complex architectures. You can choose to use as much or as little of the framework as you would like, and several of the classes (like SqlBuilder and Datum) are quite useful in their own right, even for applications that don't use DataJets.
