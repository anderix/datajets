Impedance mismatch

In writing modern software applications, there is a common problem known as impedance mismatch, which refers to the fact that object-oriented systems and relational database management systems rely on two very different conceptual paradigms. Object-oriented systems like Java are much different from relational database management systems in many ways, and designing those types of systems requires very different skills. In a typical software development scenario, time must be spent designing two different models: the object model and the data model. Developing an entirely object-oriented design is difficult, because connections must be made between the object model and the data model. The objects cannot be stored directly in the database, nor can they be loaded from the database. Because of this, object oriented systems and relational database systems are said to have an impedence mismatch.

See also: Object-Relational Impedance Mismatch

Object-relational mapping

Because of impedance mismatch between object-oriented development and relational database development, most modern software development efforts must include mappings between the two types of systems. Object-relational mapping is this process of defining the relationship betwen objects and the relational tables that store them. Object-relational mapping is not required for all systems, but it does make working with database systems from within object-oriented and object-based languages much easier. Many solutions for object-relational mapping exist today. DataJets is not a mapping framework, but a framework that enables such mapping.

See also: Object-Relational Mapping

Separation of Concerns

When designing software, it is helpful to group similar tasks together, isolating similar funcionality into seperate conceptual or physical modules. The similarities are known as concerns, and the process of isolating them is seperation of concerns. DataJets helps in this regard by isolating all database access within DataJet objects. Your DataJets will also help you seperate your business logic from display logic, further enabling modularity in your designs.
