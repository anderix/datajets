Adding Load Methods

In the last tutorial you learned how to create your own DataJets that create, update, and delete records. But most of the biggest tasks associated with database applications involves loading a record or group of records. In this tutorial I'll show you how to accomplish this with your DataJets. We'll be building on the code we wrote last time, and using the same database file. Refer to earlier tutorials for database configuration information.

To load records from the database, DataJets use static methods. You can add as many (or as few) as your applications need. You can load a single record based on its id, or you can load multiple records based on an SQL statement or SqlBuilder object.

Let's start with the method to load a single FirstDataJet object by its id:

public static FirstDataJet load(int id) throws ConnectionException, DataJetException {
	FirstDataJet dj = new FirstDataJet();
	load(id, dj, jetSet());
	return dj;
}

Here we begin by instantiating a FirstDataJet object. We pass this object, along with the id and our JetSet object, into the load method (exposed by DataJet, the parent of GenericDataJet). the load method "fills" the DataJet with the values from the database without any further action on your part. Replace the code in your main method with the following:

FirstDataJet dj = FirstDataJet.load(1);
System.out.println(dj.getTextField());

Compile and run FirstDataJet. You should see "Hello, DataJet!" on your screen.

Next we'll add the method to load all records from the database into FirstDataJet objects. Add the following:

public static FirstDataJet[] loadAll() throws ConnectionException, DataJetException {
	JetSet jset = jetSet();
	SqlBuilder sql = new SqlBuilder(jset.getTable(), jset.getFieldList());
	return (FirstDataJet[])loadMultiple(sql, new FirstDataJet[]{new FirstDataJet()}, jset);
}

In this code we are creating a local copy of our JetSet object. Although this isn't required, it makes the code a little more legible. Next, we create a SqlBuilder object based on the table and fields defined in our JetSet. The last step is to call the static loadMultiple method (exposed by DataJet), passing in our SqlBuilder, an array of FirstDataJet objects to fill, and our JetSet object. The loadMultiple method returns an array of DataJet objects, so we must cast it to an array of FirstDataJet objects.

Comment out the code in your main method and add some code to test your loadAll method:

FirstDataJet[] jets = FirstDataJet.loadAll();
for ( int i = 0; i < jets.length; i++ ) {
	System.out.println(jets[i].getTextField());
}

Compile and run FirstDataJet. You should see output from each of the records in your database.
