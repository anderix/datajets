Hello, DataJet!

Welcome to the DataJet-enabled version of the venerable "Hello, World!" To learn how to use this tutorial, please see the tutorials overview.

To get started, download the database for this tutorial, and then set up an ODBC data source named "hello" that points to it.

Next, create a class named "HelloDataJet" that contains a static main method and nothing else. Import the anderix.datajets package into your class.

import anderix.datajets.*;

In this example, we'll be creating a GenericDataJet object and calling its methods. All the code you write will be inside the main method. This is appropriate for simple actions, but typically you'll want to extend GenericDataJet to create your own DataJets. I'll cover that in a future tutorial, but for now let's stick with the basics.

We need to specify what driver and connection settings to use. We're using the JDBC-ODBC Bridge with the data source you just set up, so type the following code:

String driver = "sun.jdbc.odbc.JdbcOdbcDriver";
String url = "jdbc:odbc:hello";

In order for GenericDataJet to know what table and fields we want it to use, we're going to have to tell it! Open up the Access database and take a look at the one and only table in it. The table is called "hello," and its two fields are "id" and "text_field."

String table = "hello";
String[] fields = new String[]{"id", "text_field"};

In order to pass this information into GenericDataJet, we need to package it in a format that DataJet understands. Create a new JetSet object and use the variables we just declared as parameters in the constructor.

JetSet jetSet = new JetSet(driver, url, table, fields);

Now we have our JetSet, and we can put GenericDataJet to work. There is a record in our database with an id value of 1. Let's have GenericDataJet load that record:

GenericDataJet dj = GenericDataJet.loadJet(1, jetSet);

The loadJet method may throw a ConnectionException or a DataJetExcpetion. For the purposes of this tutorial, just add throws Exception to your main method.

Now that our record has been loaded, we can access its field values. The getFieldValue method needs to know the index of the field value you want to retrieve. This is the same as the index of the corresponding field name that you specified in the jetSet method. We want to retrieve the value of the "text_field" field, so in this case the index is 1. Print this value to the screen.

System.out.println(dj.getFieldValue(1));

That's all there is to it! Compile and run HelloDataJet. You've just created your first DataJet application!

The Identity Field

Take another look at the line of code where we defined our fields:

String[] fields = new String[]{"id", "text_field"};

Notice the location of the "id" field. This is your identity field, or primary key. Because of the importance of this field to database management systems, there are some rules to follow when it comes to using DataJets.

Rule #1: The identity field must be an automatically-incrementing integer field in your database. DataJets assumes you'll be using an integer field for the primary key, and that it will automatically increment. In our example, we're using an "AutoNumber" Access data type. Consult with your vender's documentation to see how to set this up with your database platform. This is required setup for tables you'll be connecting your DataJets to.

Rule #2: The identity field must be in the zeroth ordinal position in your field list array. Whenever you define the fields your DataJet will use, the primary key field must be in the zeroth ordinal position. If it is not, your database operations won't work correctly. There are no restrictions on the name of this field, just on its ordinal position in the array.
