Your First DataJet

In previous tutorials you learned how to write small applications that use GenericDataJet to retrieve data from a database. For most application, however, it is much more useful to extend GenericDataJet to build your own DataJet classes. This tutorial will show you how to do just that. We'll be using the same database file as we used in Hello, DataJet!. Since we made some changes to the file in the last tutorial, you might want to download a fresh copy. If you skipped those lessons, download the file and set up an ODBC data source named "hello" that points to it.

Our objective in this lesson is to build an object that will let us add, update and delete records in the "hello" table of our database. Begin by writing a class named FirstDataJet that extends GenericDataJet.

import com.datajets.*;

class FirstDataJet extends GenericDataJet {
	
}

Inside your class definition, add a static method named jetSet that returns (you guessed it) a JetSet object. Recalling the steps you took to create a JetSet object in previous tutorials, fill in the code to create and return a new JetSet that corresponds to the "hello" table in our database:

public static JetSet jetSet() {
	String table = "hello";
	String[] fields = new String[]{"id", "text_field"};
	String driver = "sun.jdbc.odbc.JdbcOdbcDriver";
	String connectionUrl = "jdbc:odbc:hello";
	return new JetSet(driver, connectionUrl, table, fields);
}

Take a quick look at the JavaDoc for GenericDataJet. There are two constructors, one that accepts an int and a JetSet, and one that just accepts a JetSet. We'll need to provide access to both of them, but now that we've defined our JetSet within a static method we can simply call that from within our constructors. Add the following:

public FirstDataJet() {
	super(jetSet());
}

public FirstDataJet(int id) {
	super(id, jetSet());
}

Now let's provide an easy way to get and set values in our record:

public void setTextField(String textField) {
	setFieldValue(1, textField);
}

public String getTextField() {
	return getFieldValue(1).toString();
}

Your first DataJet is now ready to be used. Add a main method so we can test it out:

public static void main(String[] args) throws Exception {
	FirstDataJet dj = new FirstDataJet();
	dj.setTextField("my entry");
	dj.store();
	System.out.println("Value stored in database!");
}

Compile and run the application. You should see the message telling you that the value has been stored in the database. If you open the Access file and examine the table you should see your record. For the remainder of this lesson I'm assuming that you downloaded a fresh copy of the database before you began, so the id value for the record you just created is 2. If it is not, you may need to substitute the appropriate value in your code.

Modify the code in your main method to update your record:

FirstDataJet dj = new FirstDataJet(2); //pass id value to constructor
dj.setTextField("my updated entry");
dj.store();
System.out.println("Value updated in database!");

Finally, modify the code once again to delete your record:

FirstDataJet dj = new FirstDataJet(2);
dj.remove();
System.out.println("Record removed from database!");
