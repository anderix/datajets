Introducing Datum

The word "datum" is the singular form of data. Within the DataJets framework, Datum is a class that lets you manipulate pieces of information returned by your database. This tutorial is an introduction to the Datum type and its members. It will build on the code created in Hello, DataJet! and we'll also be using the same database. Don't forget that you'll need an ODBC data source named "hello" that points to the database.

Let's explore Hello, DataJet! a little more. In your code, take another look at the last line in the main method:

System.out.println(dj.getFieldValue(1));

What do you suppose the class of the object returned by getFieldValue is? Yep, you guessed it: it's Datum. Now, before you tell me that Datum doesn't do you any good because you need it to return an int or a String, let me explain. Datum is a wrapper around Object, and the object it contains is actually of the type returned by your database for the specific field. For example, if your field value is stored in a field that is defined for text data (say, varchar) then the object inside Datum is really a String. You just need to convert your Datum into a String.

There are two basic categories of methods that Datum provides. The first is a group of methods that follow the is* naming convention and return a boolean value. These methods, as you might expect, are for testing the type of the object inside Datum to determine if it is of the specified type. Rewrite the last line of code in HelloDataJet.java so that it looks like this:

Datum d = dj.getFieldValue(1);
System.out.println(d.isString());

Compile and run your application. You'll see that isString returns true, because it contains a String datum. Try testing several other types:

System.out.println(d.isInt());
System.out.println(d.isBoolean());

If you open up your database file and change the data type of the field, you'll notice that the type of the object returned also changes. By returning object of type Datum in getFieldValue, you will always have a consistent way of working with the values, regardless of type. In practice, you will usually know the type of data you are expecting, and you will just need to convert it appropriately.

Which brings us to the second category of methods: those that are used for conversion. These methods follow the to* naming convention. If you changed the field type in your database, return it to the Text data type and change the value in the field to 1 (that's the number one, not the letter l). Then add:

System.out.println(d.toString());
System.out.println(d.toInt());
System.out.println(d.toBoolean());

Compile and run. The object type is once again String, but all the to* methods were able to return values. None of Datum's methods throw exceptions; if a piece of data cannot be converted, a default value is returned. Consult the JavaDoc for specific default values. Please be careful when converting values to a type other than its database type. You don't want to mistake default values for real data! Use the is* methods if you are unsure of the type.

Several of the classes within the com.datajets package are useful apart from the DataJets framework. Datum is one of those classes. For example, I've used it when retrieving values from a querystring or form post in JSP. I'm sure you'll think of other places where it can help out. Use Datum and you'll have a consistent way to perform conversions between types.
